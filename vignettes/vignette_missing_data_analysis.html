stackr missing data visualization analysis

Thierry Gosselin

2016-06-22

Objectives

Learn how to visualize missing genotypes in your genomic dataset with the function stackr::missing_visualization (time = 15 min).

Workflow

The function missing_visualization in stackr uses various genomic input files and conduct identity-by-missingness analyses (IBM) using Principal Coordinates Analysis (PCoA), also called Multidimensional Scaling (MDS). Figures and summary tables of missing information at the marker, individual and population level are generated. Below, the simplest form of the function to get results … fast! More options are available, see function documentation.

Get the data and create a strata file

  1. Dataset: in this example, we use the data in Ferchaud and Hansen (2015 and 2016) paper.

    writeBin(httr::content(httr::GET("http://datadryad.org/bitstream/handle/10255/dryad.97237/sticklebacks_Danish.vcf?sequence=1"), "raw"), "example_vcf2dadi_ferchaud_2015.vcf")
  2. We need to generate a strata file necessary to make the vcf population-wise. The strata file is a tab-delimited file or object, in the global environment, with a minimum of 2 columns: INDIVIDUALS and STRATA. The STRATA and remaining columns can be any hierarchical grouping.

    • Because the population id is contained in the name of the individuals and easily extracted with string command, we use the function stackr::individuals2strata. r # here is one individual id: Hadsten10_q_75.align # with pop starting at position 1 and to keep only 3 characters (necessary to discriminate populations from each other) we stop at... 3

    • For info on the function use: ?individuals2strata. Don’t have this kind of names… you can also make the strata file by hand, the old fashion way.

    • For your data, you might not need to make a strata file if you have already run on your data, the strata file is similar to a stacks population map file, make sure you have the required column names (INDIVIDUALS and STRATA).

    • We will extract the id from the first line of the vcf file. If the individuals were in a single column in a file (e.g. “individuals.txt”), make sure the column is named INDIVIDUALS and use the file directly in the function stackr::individuals2strata.

    • We could put all the lines below in one call, but for clarity we’ll break down the steps:

      # Isolate the first line

The function generates by default the IBM-PCoA plot, the heatmap plot and several plots showing the distribution of missing genotypes by individuals, pop and markers. Blacklists of individuals are included in the list ibm and written to the working directory (defaults from 10% to 70%, that automatically stops if no individuals at the threshold is found, can be changed with ind.missing.geno.threshold argument). Tables with summary missing information along the vectors of eigenvalues of the principal coordinates analysis are also part of the list output. All these objects are accessed with $.

For the second run, we are going to select all the groupings in the strata file. This will allow us to get the IBM-PCoA plot for each groupings automatically.

ibm <- stackr::missing_visualization(
data = "example_vcf2dadi_ferchaud_2015.vcf", # VCF file, but see function documentation for other options
strata = "strata.stickleback.tsv", # 4 columns: INDIVIDUALS, STRATA, POP_TYPE, ECOTYPE
strata.select = c("POP_ID", "POP_TYPE", "ECOTYPE") # STRATA is changed to POP_ID automatically 
)

Interpretation

Do you see patterns in your plots that provides insight about the relationships that missing values might have with other variables (inspired from r4ds).

If you see a pattern, ask yourself:

  • Is the pattern due to coincidence (i.e. random chance)?

  • Could you describe the relationship in the pattern ?

  • How strong is the relationship implied by the pattern?

  • What other variables might affect the relationship?

  • Does the relationship change if you look at individual subgroups of the data?

  • Do you think the pattern observed in the data could impact the clustering analysis?

Arguments

  • use strata.select to select columns from the strata file to generate PCoA-IBM plots. If you have several columns to test e.g. library, sequencer, sequencing lanes, sampling sites, populations, use inside a string to get all of them!
  • use distance.method to explore other distance metric used by the function dist

Filtering strategies

  • explore the impact of different filtering parameters on missing genotypes, with available arguments: e.g. snp.ld, common.markers, maf.thresholds arguments to quickly generate the figures.
  • test your own whitelist of markers with whitelist.markers
  • use the blacklist of individuals created with the function to manage missing data inside filtering pipelines
  • excluding/including individuals, populations and markers to test the impact of filtering on polymorphism discovery
  • ready to test missing data imputations ? use most of stackr functions with the imputations arguments…
